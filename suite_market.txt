- Déconnexion dans onglet "Paramètres" de /account
### Tables Supabase à créer
```sql
-- Profils utilisateurs (extension de auth.users)
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  first_name TEXT,
  last_name TEXT,
  phone TEXT,
  city TEXT,
  country TEXT DEFAULT 'Bénin',
  created_at TIMESTAMPTZ DEFAULT now()
);
-- Adresses de livraison
CREATE TABLE addresses (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  label TEXT,           -- ex: "Maison", "Bureau"
  full_name TEXT,
  city TEXT,
  district TEXT,
  phone TEXT,
  is_default BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);
-- Commandes
CREATE TABLE orders (
  id TEXT PRIMARY KEY,  -- format: 'TL-20260114-001'
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  status TEXT CHECK (status IN ('pending','processing','shipped','delivered')),
  items JSONB,          -- snapshot cart items
  total INTEGER,        -- en XOF
  address JSONB,        -- snapshot adresse
  tracking_number TEXT,
  payment_method TEXT,  -- 'mtn_momo' | 'moov_money' | 'wave'
  created_at TIMESTAMPTZ DEFAULT now()
);
-- Wishlist
CREATE TABLE wishlists (
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  product_id TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  PRIMARY KEY (user_id, product_id)
);

RLS à activer

    addresses : ALL WHERE user_id = auth.uid()

    orders : SELECT/INSERT WHERE user_id = auth.uid()

    wishlists : ALL WHERE user_id = auth.uid()

PHASE 7 — PAIEMENT MOBILE MONEY (Edge Functions)
Architecture

Frontend → Supabase Edge Function → API MTN MoMo / Moov Money / Wave

Edge Functions à créer

    supabase/functions/initiate-payment/index.ts
    Input : { orderId, amount, phone, method: ‘mtn’|’moov’|’wave’ }
    Output : { paymentId, status, redirectUrl? }

    supabase/functions/verify-payment/index.ts
    Input : { paymentId, method }
    Output : { status: ‘success’|’pending’|’failed’ }

    supabase/functions/payment-webhook/index.ts
    verify_jwt: false (webhook externe)
    Met à jour orders.status via service_role

Secrets Supabase

MTN_MOMO_API_KEY=xxx
MTN_MOMO_SUBSCRIPTION_KEY=xxx
MTN_MOMO_ENV=sandbox|production
MOOV_API_KEY=xxx
WAVE_API_KEY=xxx
ESCROW_HOLD_DAYS=7

Flux Escrow simulé

    Checkout → initiate-payment → order.status = ‘pending’

    Paiement confirmé → order.status = ‘processing’

    Vendeur envoie au Hub → order.status = ‘shipped’

    Acheteur confirme réception → release-escrow → order.status = ‘delivered’
    (ou après ESCROW_HOLD_DAYS jours automatiquement)
