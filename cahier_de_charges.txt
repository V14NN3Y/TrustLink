Gestion d’État

    Panier : useCart.ts — Hook custom avec useState + localStorage pour persistance

    Wishlist : useWishlist.ts — même pattern

    Pas de state global (pas de Redux, pas de Zustand) — volontairement léger

    Pas de fetching asynchrone — données 100% mockées (Phase 1–2)

Animations

    Hero : slide par opacity-0/100 + transition-opacity duration-700

    Badge support flottant : animate-bounce

III. LOGIQUE D’INTERFACE (UI/UX)
Layout Général

    Conteneur maximal :max-w-[1200px] mx-auto px-4 md:px-6 sur tout le site

    Structure page :min-h-screen flex flex-col → Header (fixed) + main (flex-1) + Footer

    Padding top compensatoire :pt-16 md:pt-20 pour compenser le header fixed (h-16 mobile / h-20 desktop)

Header (composant Header.tsx)

Comportement :

    Fixed en haut (fixed top-0 left-0 right-0 z-50)

    Fond blanc dès le début (contrairement à un hero en transparency), shadow-sm activé au scroll > 10px via window.scrollY

    Layout desktop : Logo | Search bar (flex-1, max-w-xl) | [Cart icon + Account icon]

    Layout mobile : Logo | [Cart + Account + Hamburger] puis search bar sur deuxième ligne

Logique recherche :form onSubmit → navigate('/?search=...') via encodeURIComponent. La page d’accueil lit le param search depuis useSearchParams() et filtre les produits côté client.
Barre de Catégories (composant CategoryBar.tsx)

    Sticky sous le header (top-16 md:top-20)

    6 pills horizontaux scrollables : Mode / Beauté / High-Tech / Auto / Maison / Sport

    Pill actif : bg-[#125C8D] text-white, inactif : bg-white text-[#111827] border

    Clic → navigate('/?category=...') — la catégorie “Tout” vide le param

Hero Banner (composant HeroBanner.tsx)

    3 slides avec rotation automatique toutes les 5 secondes (setInterval)

    Transition : opacity-0 / opacity-100 avec duration-700

    Overlay : bg-gradient-to-r from-[#0E3A4F]/85 via-[#0E3A4F]/60 to-transparent

    Contenu centré à gauche (max-w-lg)

    Structure contenu : Badge orange pill → H1 whitespace-pre-line → Description → 2 boutons côte à côte

    Dots de pagination : pill w-6 bg-[#FF6A00] si actif, w-2 bg-white/50 si inactif

    Flèches : bg-white/20 hover:bg-white/40 cercles 9/10 taille

ProductCard (composant ProductCard.tsx)

Structure :Link to="/product/:id" wrappant une div flex-col

    Image : hauteur fixe h-52, object-cover object-top, hover scale-105

    Badge promo orange : absolute top-2 left-2 — si product.discount existe

    Bouton wishlist cœur : absolute top-2 right-2 — toggle ri-heart-fill text-[#FF6A00] vs ri-heart-line text-[#6B7280]

    Étoiles : 5 icônes Remix, full/half/empty selon rating

    Prix : text-[#125C8D] font-bold + prix barré optionnel

    Bouton “Ajouter au panier” : toujours visible (pas au hover) — fond #E1F0F9 → hover #125C8D blanc — feedback vert “Ajouté !” 1.5s

ProductGrid (composant ProductGrid.tsx)

IV. COMPOSANTS & LOGIQUE DÉTAILLÉE
useCart (hook)

// État : CartItem[] dans localStorage clé "trustlink_cart"
// Méthodes exposées :
addItem(item)          // ajoute ou incrémente quantité
removeItem(productId)  // supprime un article
updateQuantity(id, qty) // change la quantité
clearCart()            // vide le panier
totalItems             // nombre total d'articles (somme des qty)
totalPrice             // prix total en XOF

useWishlist (hook)

// État : string[] (product IDs) dans localStorage clé "trustlink_wishlist"
toggle(productId)       // ajoute ou retire
isWishlisted(productId) // boolean
items                   // string[]

FloatingSupportButton

    Position : fixed bottom-6 right-6 z-50

    Par défaut : visible avec label “Support TrustLink” + animation animate-bounce

    Bouton × pour fermer (useState showLabel)

    Couleur : bg-[#125C8D] icône chat blanc

Page /checkout — Tunnel en 3 étapes

    Étape 1 : Formulaire adresse livraison (nom, ville, téléphone, quartier)

    Étape 2 : Récapitulatif + explication Escrow + validation paiement

    Étape 3 : Confirmation avec numéro de suivi généré aléatoirement (TL-XXXXXXXX)

    Logique : currentStep (1/2/3), navigation via boutons “Continuer” / “Retour”

    Pas de soumission réelle — simulation pure

Page /account — Layout sidebar

    Bandeau profil : fond sombre, avatar, badge Premium, stats (commandes/total/note)

    Sidebar gauche : 4 onglets (Mes Commandes / Mon Profil / Mes Adresses / Paramètres)

    Commandes : filtres par statut (Tout / En traitement / Expédiée / Livrée) + cards détaillées avec bouton “Confirmer réception”

    Profil : grille de champs éditables avec feedback “Sauvegardé !”

    Paramètres : toggles notifications, sécurité 2FA, zone de danger

V. MODÈLE DE DONNÉES MOCK (suite)
Interface Product (complète)

{
  id: string
  name: string
  category: 'mode' | 'beaute' | 'hightech' | 'auto' | 'maison' | 'sport'
  price: number              // en XOF entier (ex: 28500)
  originalPrice?: number     // si en promo
  discount?: number          // % affiché dans le badge orange
  images: string[]           // min 1 URL, max 4 (galerie fiche produit)
  rating: number             // float entre 4.4 et 4.9
  sales: number              // critère tri "Populaire"
  variants: {
    sizes: string[]          // vide [] si non applicable
    colors: { name: string; hex: string }[]
  }
  description: string        // 1-2 phrases
  seller: string             // nom de la boutique Nigeria
  inStock: boolean
  isNew?: boolean            // badge "Nouveau" (non utilisé encore en v1)
  isFeatured?: boolean       // pour section "À la une" future
  wishlisted?: boolean       // état initial wishlist (mock only)
}

Interface CartItem (dans useCart.ts)

{
  productId: string
  name: string
  image: string              // images[0] du produit
  price: number
  quantity: number
  selectedSize?: string
  selectedColor?: string
}

Interface Order (dans orders.ts)

{
  id: string                 // format: 'TL-20260114-001'
  status: 'pending' | 'processing' | 'shipped' | 'delivered'
  items: {
    productId: string
    name: string
    image: string
    price: number
    quantity: number
  }[]
  total: number              // en XOF
  address: {
    name: string
    city: string
    district: string
    phone: string
  }
  trackingNumber?: string    // format: 'TL-XXXXXXXX'
  createdAt: string          // ISO date string
}

Règles impératives routing :

    BrowserRouter avec basename={__BASE_PATH__} dans App.tsx (variable injectée par Vite)

    element toujours un JSX (<Component />), jamais lazy() direct

Formatage monétaire (fonction utilitaire)

function formatPrice(price: number): string {
  return new Intl.NumberFormat('fr-FR').format(price) + ' FCFA';
}
// Résultat : "28 500 FCFA"
